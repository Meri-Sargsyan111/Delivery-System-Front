How It Works

The delivery workflow, from order creation to completion

  1. Create Order

    A new delivery order is created through the Angular frontend.

  2. Order Processing

    Backend microservices process the order and manage its delivery status.

  3. Courier Processing

    The courier service handles courier assignment and delivery processing.

  4. Kafka Events

    Services communicate asynchronously through Apache Kafka events.

  5. Live Courier Location

    Courier location updates are streamed in real time as the delivery progresses.

  6. WebSocket / STOMP

    The Angular frontend receives those location updates through a WebSocket/STOMP connection.

  7. Google Maps Live Tracking

    The courier marker moves on Google Maps as new position updates arrive.

  8. Notifications

    Real-time notifications inform the user about relevant delivery events.

  9. Delivery Completed

    The order status is finalized and its tracking history reflects the completed delivery.
